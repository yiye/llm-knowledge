## ✍️ 撰写规则

### 层次递进原则

- **foundations/**：假设零基础，讲清楚
- **core-concepts/**：深入本质，解释"为什么"
- **advanced/**：聚焦实践，展示"怎么做"

### 系统性要求

- **前后呼应**：概念首次出现简单说，后续深入展开
- **引用关联**：用相对路径引用其他文档，形成知识网络
  ```markdown
  参考 [点积与相似度](../foundations/L-2-dot-product-similarity.md)
  ```

### 必备要素

- 核心概念（定义）+ 为什么重要（问题）+ 技术演进（时间线）+ 实践案例（真实）+ 参考资料（可追溯）

## 🎯 创建与更新

### 创建新文档

✅ 新的核心概念（Transformer、MoE）、重要技术演进（RLHF → DPO）、里程碑事件
❌ 零散技巧、项目代码分析、产品创意（放其他目录）

### 更新现有文档

✅ 技术有新进展 → 追加章节，标注时间
✅ 发现新案例 → 补充到"实践案例"
✅ 修正错误 → 直接修改，提交说明

### 拆分标准

文档超过 **500 行**或包含 **3+ 独立主题** → 考虑拆分
